进程

Jan 9, 2017 | Hits

代码段，数据段，打开的文件，内核内部数据，映射的内存地址空间，多个执行的线程等资源的合集，用进程来描述。

进程描述符

内核通过slab分配器分配进程描述符task_struct，每个进程的所有资源都用它记录。用户态进程的task_struct 创建之后会加入链表init_task.tasks中，init_task为全局变量，是系统的用户态第一个进程。

  • thread_info: 保存在内核栈或者寄存器中，描述了进程的基本信息。
  • mm_struct: 指向内存区描述符的指针。
  • tty_sturct: 与进程相关的tty。
  • files_sturct: 指向文件描述符的指针。
  • signal_sturct: 所接收的信号。
  • state：进程状态。

进程树

整个Linux系统的所有进程是一个树形结构。树根是系统自动构造的，即在内核态下执行的0号进程，它是所有进程的祖先。由0号进程（idle）创建1号（init）进程（内核态），1号负责执行内核的部分初始化工作及进行系统配置，并创建若干个用于高速缓存和虚拟主存管理的内核线程。随后，1号进程调用execve（）运行可执行程序init，并演变成用户态1号进程，即init进程。init进程是所有用户态进程的祖先。

内核线程

内核线程只运行在内核态，可以认为是没有用户态代码的特殊进程？内核线程的创建最终都是调用的do_fork,没有用户态地址空间。

  • 0号进程idle：当内核被加载到内存后，完成内核初始化的函数start_kernel()从无到有的创建一个内核线程swap，其PID为0，也叫闲逛进程；进程0执行的是cpu_idle()函数，当就绪队列中再没有其他进程时，闲逛进程就会被调度程序选中，以此来省电，减少热量的产生。
  • 1号进程init：init 进程，由0进程创建，完成系统的初始化后，转变为用户态进程，是系统中所有其它用户进程的祖先进程

函数调用栈

每个用户态进程拥有自己的用户态栈和内核态栈，进程陷入内核之后使用内核栈，内核栈的栈顶或栈底（不影响栈使用）记录了struct thread_info,此结构体记录了进程描述符指针， 宏current获取进程描述符就是通过计算内核栈的偏移得到的（寄存器不多的cpu架构，如x86）。

union thread_union {
    struct thread_info thread_info;
    unsigned long stack[2048]; /* 栈大小为8k */
} /* x86一般为8k，联合体按最大需要的内存分配 */

进程的用户态线程的调用栈都是线程私有的资源，进程陷入内核之后使用的内核栈也是私有，线程之间是并发执行的，私有堆栈保证函数调用不受其他线程的影响，加锁效率太低。线程堆栈不能动态增长，大小是固定的，是从进程的地址空间中map出来的一块内存区域。

x86架构的处理器中断栈是独立于进程内核栈的，所有软中断共享一个软中断栈，所有的硬中断共享一个硬中断，ARM上进程栈和中断栈则是共享的。

内核线程的栈是每个内核线程都有一个私有的函数堆栈

Git learn

Dec 19, 2016 | Hits

文章记录下常用的git命令，等积累到一定程度了，可以用一张图简单的展示出来.

Com Git cmd

  • git checkout master
  • git commit -a
  • git log //也许该查查 怎么查单个文件的修改记录
  • git reset HEAD^//撤销上一次提交
  • git pull -r //git fetch + git rebase的合体
  • git push master:master // 本地拉一个分支也可以git push origin <分支名> 上传到远程
  • git branch -a
  • git status //无论什么时候这个命令可以知道下一步该干什么，包括冲突解决
  • git merge //还得好好研究下

GIT CMD

  • git branch -vv：查看当前分支的上游分支
  • git cherry-pick [comit-id] ： 可以从其他分支优雅的捡起你想要的任何提交
  • git rebase -i: 可以将本次多次提交合并为一次提交，甚至更多交互rebase
  • git stash / git stash pop: 本地修改压入栈，可随存随取。取回时，可能要处理冲突。
  • git checkout [file]： 丢弃file的修改
  • git clean -df： 丢弃未跟踪的文件
  • git add ./: 当前目录下添加所有需要添加的文件

杂七杂八

  • 本地合并多个commit: 先git reset HEAD~n, n表示从HEAD回退多少个提交; 然后git commit -a一次提交.
  • 远程回退一次提交: git revert HEAD^(commit ID),此次操作之前和之后的commit都会保留，把这次撤销作为一次最新的提交，不影响之前提交的内容.
  • git commit --amend误操作的回退：不小心提交代码到本地的他人的commit A中，先直接git reset HEAD^回退A,git stash 保存A和自己的修改；rebase 之后 git stash pop 完美回退。
  • 远程回退多次提交: 远程回退一次提交，can not merge 想想原因
  • 冲突处理：cherry-pick、rebase、stash这些命令执行之后，看下提示，提示会提醒你解决冲突之后你的下一步操作
  • git reset: –soft 回到commit之前的状态, –mixed(默认) 回到git add执行之前的状态， –hard, 代码都没了,但是可以找回来, git会保留对象一段时间(几个月).
  • git reset --hard的找回：1. git log -g 找到回退的commit id, cherry-pick过来; 2. 找到commit id之后，git branch commitID.

后记

公司前一段时间项目代码管理要求强制从svn切换到git工具，这个适应过程花了不少功夫。一开始使用git，除了下载代码速度比svn快之外，其它方面git用起来也是非常麻烦。任何一个事物或者是人，只有真正认识和熟悉了，你才能全面的去评价和认识他。文章简单的记录下常用的和一些冷门但是非常有用的git命令。事实上，如果能用一张图将git整体框架和概念展示出来，那就再好不过了。

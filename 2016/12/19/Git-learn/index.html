Git learn

Dec 19, 2016 | Hits

公司前一段时间项目代码管理要求强制从svn切换到git工具，这个适应过程花了不少功夫。一开始使用git，除了下载代码速度比svn快之外，其它方面git用起来也是非常麻烦。任何一个事物或者是人，只有真正认识和熟悉了，你才能全面的去评价和认识他。文章简单的记录下常用的和一些冷门但是非常有用的git命令。事实上，如果能用一张图将git整体框架和概念展示出来，那就再好不过了。

Com Git cmd

git checkout master
git commit -a 
git log         //也许该查查 怎么查单个文件的修改记录
git reset HEAD^//撤销上一次提交
git pull -r  //git fetch + git rebase的合体
git push master:master
git branch -a
git status  //无论什么时候这个命令可以知道下一步该干什么，包括冲突解决
git merge   //还得好好研究下

–

冷门Git CMD

-git branch -vv：查看当前分支的上游分支
-git cherry-pick [comit-id] ： 可以从其他分支优雅的捡起你想要的任何提交
-git rebase -i: 可以将本次多次提交合并为一次提交，甚至更多交互rebase
-git stash / git stash pop: 本地修改压入栈，可随存随取。取回时，可能要处理冲突。
-git checkout [file]： 丢弃file的修改
-git clean -df： 丢弃未跟踪的文件
-git add ./: 当前目录下添加所有需要添加的文件
